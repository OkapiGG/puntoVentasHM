PANTALLA 1: LOGIN (INICIO DE SESIÓN)

Esta pantalla permite el acceso al sistema según el rol del usuario.

Elementos de la pantalla:

Campo de usuario
Campo de contraseña
Botón iniciar sesión

Opcional:

Recordar sesión
Mensaje de error si los datos son incorrectos

Roles posibles:

Administrador / gerente
Encargado
Mesero
Caja

El acceso determinará las funciones disponibles dentro del sistema.

PANTALLA 2: PANEL PRINCIPAL (DASHBOARD)

Esta pantalla muestra un resumen general del funcionamiento del restaurante durante el día.

Elementos:

Resumen de ventas del día
Número de pedidos realizados
Ventas por tipo de pago
Estado de caja
Productos más vendidos
Promociones activas

También permite acceso rápido a:

Nuevo pedido
Caja
Reportes

PANTALLA 3: NUEVO PEDIDO (POS PRINCIPAL)

Esta es la pantalla principal del sistema donde se registran las ventas.

La interfaz se divide en tres secciones.

Sección izquierda (navegación):

Nuevo pedido
Mesas
Productos
Combos
Promociones
Caja
Reportes
Configuración

Sección central (selección de productos):

Categorías de productos:

Hamburguesas
Snacks
Postres
Bebidas
Combos
Promociones

Cada producto se muestra en una tarjeta que contiene:

Nombre
Precio
Imagen
Botón agregar

Sección derecha (resumen del pedido):

Lista de productos agregados
Cantidad
Precio por producto
Ingredientes extras
Subtotal
Costo de envío si aplica
Total del pedido

Botones disponibles:

Eliminar producto
Modificar extras
Proceder a pago

PANTALLA 4: SELECCIÓN DE INGREDIENTES EXTRA

Cuando el usuario selecciona una hamburguesa aparece una ventana para agregar extras.

Opciones:

Queso extra
Tocino extra
Carne extra
Salsa extra

Cada ingrediente debe mostrar su costo adicional.

También debe permitir eliminar extras.

PANTALLA 5: SELECCIÓN DE TIPO DE PEDIDO

Antes de finalizar el pedido se debe seleccionar el tipo de servicio.

Opciones:

Consumir en el local (mesa)
Pedido rápido para recoger en sucursal
Servicio a domicilio

Si es servicio a domicilio se debe agregar el costo de envío.

PANTALLA 6: GESTIÓN DE MESAS

Esta pantalla muestra el mapa de mesas del restaurante.

Elementos:

Vista gráfica de mesas
Estado de cada mesa (libre, ocupada)
Número de mesa

Opciones disponibles:

Asignar mesa a un pedido
Asignar mesero a la mesa
Ver pedidos activos

PANTALLA 7: PANTALLA DE PAGO

Permite finalizar el pedido y registrar el pago.

Métodos de pago disponibles:

Efectivo
Terminal bancaria (tarjeta)
Transferencia

Si es efectivo debe permitir diferentes monedas:

Pesos mexicanos
Dólares
Quetzales

Debe mostrar:

Total a pagar
Dinero recibido
Cambio

PANTALLA 8: ADMINISTRACIÓN DE PRODUCTOS

Permite administrar el menú del restaurante.

Categorías disponibles:

Hamburguesas
Snacks
Postres
Bebidas

Opciones disponibles:

Agregar producto
Editar producto
Eliminar producto
Asignar precio
Asignar imagen

También debe permitir definir:

Ingredientes extra disponibles
Costo de cada ingrediente extra

PANTALLA 9: ADMINISTRACIÓN DE COMBOS

Permite gestionar los combos del restaurante.

Tipos de combos:

Combos fijos
Combos por promoción

Opciones disponibles:

Crear combo
Editar combo
Eliminar combo

Cada combo debe incluir:

Productos incluidos
Precio del combo

PANTALLA 10: PROMOCIONES

Permite crear promociones temporales.

Ejemplos:

Combo cumpleaños
Promoción día de las madres

Cada promoción debe tener:

Nombre
Productos incluidos
Precio promocional
Fecha de inicio
Fecha de finalización

Las promociones pueden:

Activarse manualmente
Desactivarse automáticamente al terminar la fecha.

PANTALLA 11: CONTROL DE CAJA

Permite gestionar el dinero de la caja registradora.

Funciones:

Apertura de turno
Registro de dinero inicial
Registro de ventas
Registro de retiros de dinero

Debe mostrar:

Dinero inicial en caja
Dinero por ventas
Dinero retirado

PANTALLA 12: RETIRO DE DINERO

Permite retirar dinero de la caja por seguridad.

Condiciones:

Solo el gerente puede autorizar retiros.

Debe registrar:

Cantidad retirada
Motivo del retiro
Usuario que autorizó el retiro

También debe existir un límite máximo permitido de dinero en caja.

PANTALLA 13: CANCELACIÓN DE PEDIDOS

Permite cancelar pedidos registrados.

Solo el encargado o gerente puede realizar esta acción.

Debe registrar el motivo de cancelación.

Motivos posibles:

Error en el pedido
Cancelación del cliente
Problema de pago

PANTALLA 14: PRE CORTE

Permite revisar un reporte temporal de las ventas del día sin cerrar el turno.

Debe mostrar:

Ventas totales
Productos vendidos
Métodos de pago utilizados

PANTALLA 15: CORTE FINAL DEL DÍA

Se realiza al finalizar la jornada laboral.

Debe mostrar:

Dinero inicial en caja
Total de ventas del día
Retiros realizados
Ventas por tipo de pago

También debe incluir el detalle de productos vendidos:

Tipos de hamburguesas vendidas y cantidad
Snacks vendidos
Bebidas vendidas
Ingredientes extras vendidos
Pedidos a domicilio

PANTALLA 16: REPORTES

Permite generar reportes para administración y contabilidad.

Tipos de reportes:

Reporte diario
Reporte semanal
Reporte mensual

Información mostrada:

Total de ventas
Productos más vendidos
Ventas por tipo de pago
Resumen financiero

Descripción de pantallas del sistema. El sistema es un POS para restaurante que permite gestionar pedidos, mesas, productos, caja, promociones y reportes. El sistema debe ser escalable para permitir agregar más productos o categorías en el futuro. Utiliza una interfaz moderna con colores naranja, amarillo y blanco, sin logotipos ni marcas para que pueda utilizarse en distintos restaurantes.
